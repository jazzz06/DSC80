DSC 80 Project 04: Cooking Time vs Recipe Ratings

Step 1: Introduction

Dataset Overview

This project uses the Recipes and Ratings dataset, which includes two CSV files:

  • RAW_recipes.csv: 83,782 rows (one row per recipe)
  • interactions.csv: 731,927 rows (one row per user–recipe interaction, including ratings and reviews)

The dataset is useful for understanding what recipe characteristics (like cooking time, complexity, or nutrition) are associated with higher user ratings.

Research Question

What is the relationship between cooking time (minutes) and the average rating (avg_rating) of a recipe?

This question matters because cooking time affects how practical a recipe is for different users. If longer recipes tend to be rated higher (or lower), that insight can help people choose recipes and help recipe writers design recipes that are both feasible and well-received.

Relevant Columns

From RAW_recipes.csv:

  • id: unique recipe identifier (used to merge with interactions)
  • name: recipe name (useful for interpretation / hover text)
  • minutes: total cooking time in minutes (main feature)

From interactions.csv:

  • recipe_id: recipe identifier (matches id)
  • rating: user rating (we will treat rating = 0 as missing)

Derived Column

To connect cook time to ratings, I will create avg_rating: the mean rating for each recipe, computed by grouping interactions.csv by recipe_id (after replacing rating == 0 with missing values), then merging onto RAW_recipes.csv.
